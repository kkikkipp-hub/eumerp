## ERD 설명

1. **주문(Order)**
   - **고객사(Customer)**와 다대일(N:1) 관계: 하나의 고객사는 여러 주문을 할 수 있다.
   - **주문 상태(OrderStatus)**와 일대다(1:N) 관계: 하나의 주문은 여러 상태 변경 기록을 가질 수 있다.
   
2. **재고(Inventory)**
   - **품목(Item)**과 일대일(1:1) 관계: 각 품목은 하나의 재고 정보를 가진다.
   - **입출고 이력(InventoryHistory)**과 일대다(1:N) 관계: 하나의 재고는 여러 입출고 이력을 가진다.

3. **정산/회계(Sales, Purchases)**
   - **거래처(Vendor)**와 다대일(N:1) 관계: 하나의 거래처는 여러 매출 및 매입 기록을 가질 수 있다.

4. **사용자(User)**
   - **역할(Role)**과 다대다(N:M) 관계: 하나의 사용자는 여러 역할을 가질 수 있으며, 하나의 역할은 여러 사용자에게 부여될 수 있다.

## 전체 CREATE TABLE SQL

```sql
CREATE TABLE customer (
    customer_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    contact_info VARCHAR(255)
);

CREATE TABLE order_status (
    status_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

CREATE TABLE orders (
    order_id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT NOT NULL,
    order_date DATETIME NOT NULL,
    item_id INT NOT NULL,
    quantity INT NOT NULL,
    status_id INT NOT NULL,
    notes TEXT,
    urgent_flag BOOLEAN DEFAULT FALSE,
    modified_at DATETIME,
    canceled_at DATETIME,
    FOREIGN KEY (customer_id) REFERENCES customer(customer_id),
    FOREIGN KEY (status_id) REFERENCES order_status(status_id)
);

CREATE TABLE item (
    item_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT
);

CREATE TABLE inventory (
    item_id INT PRIMARY KEY,
    current_quantity INT NOT NULL,
    warehouse_location VARCHAR(255),
    FOREIGN KEY (item_id) REFERENCES item(item_id)
);

CREATE TABLE inventory_history (
    history_id INT AUTO_INCREMENT PRIMARY KEY,
    item_id INT NOT NULL,
    quantity INT NOT NULL,
    transaction_date DATETIME NOT NULL,
    user_id INT NOT NULL,
    FOREIGN KEY (item_id) REFERENCES item(item_id)
);

CREATE TABLE vendor (
    vendor_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    contact_info VARCHAR(255)
);

CREATE TABLE sales (
    sales_id INT AUTO_INCREMENT PRIMARY KEY,
    month DATE NOT NULL,
    amount DECIMAL(10, 2) NOT NULL,
    vendor_id INT NOT NULL,
    FOREIGN KEY (vendor_id) REFERENCES vendor(vendor_id)
);

CREATE TABLE purchases (
    purchase_id INT AUTO_INCREMENT PRIMARY KEY,
    month DATE NOT NULL,
    amount DECIMAL(10, 2) NOT NULL,
    vendor_id INT NOT NULL,
    FOREIGN KEY (vendor_id) REFERENCES vendor(vendor_id)
);

CREATE TABLE role (
    role_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

CREATE TABLE user (
    user_id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(255) NOT NULL
);

CREATE TABLE user_role (
    user_id INT NOT NULL,
    role_id INT NOT NULL,
    PRIMARY KEY (user_id, role_id),
    FOREIGN KEY (user_id) REFERENCES user(user_id),
    FOREIGN KEY (role_id) REFERENCES role(role_id)
);
```

## 각 테이블별 설명

1. **customer**: 고객사 정보를 저장하는 테이블로, 주문과 연결됩니다.
2. **order_status**: 주문의 상태를 정의하는 테이블로, 주문 상태 변경을 관리합니다.
3. **orders**: 고객의 주문 정보를 저장하며, 주문 상태와 고객사와 연결됩니다.
4. **item**: 재고 관리 대상 품목 정보를 저장합니다.
5. **inventory**: 각 품목의 현재 재고 수량과 창고 위치를 관리합니다.
6. **inventory_history**: 품목의 입출고 이력을 기록합니다.
7. **vendor**: 거래처 정보를 저장하며, 매출 및 매입과 연결됩니다.
8. **sales**: 월별 매출 정보를 저장합니다.
9. **purchases**: 월별 매입 정보를 저장합니다.
10. **role**: 사용자 역할 정보를 저장합니다.
11. **user**: 시스템 사용자 정보를 저장합니다.
12. **user_role**: 사용자와 역할 간의 관계를 정의합니다.

## 데이터 마이그레이션 고려사항

1. **기존 엑셀 데이터 마이그레이션**: 엑셀 데이터를 데이터베이스로 이전할 때 데이터 정합성을 유지해야 합니다. 특히, 거래처 코드 체계와 주문 상태 등의 데이터가 정확히 매핑되어야 합니다.
2. **데이터 정제**: 중복 데이터 제거 및 필수 데이터 누락 여부를 확인해야 합니다.
3. **테스트 데이터**: 마이그레이션 후 테스트 데이터를 통해 시스템의 정상 동작을 검증해야 합니다.
4. **백업 및 복구 계획**: 데이터 마이그레이션 중 데이터 손실을 방지하기 위한 백업 및 복구 계획이 필요합니다.
