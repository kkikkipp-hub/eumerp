  - 화면 흐름: Step 1: 고객사 로그인 → Step 2: 주문 페이지 이동 → Step 3: 주문 접수/수정/취소 선택 → Step 4: 주문 정보 입력 및 저장
  - 비즈니스 로직: 주문 접수 시 주문 상태는 '접수'로 설정된다. 수정 시 기존 주문 정보를 업데이트하며, 취소 시 주문 상태는 '취소'로 변경된다.
  - 데이터 항목: 주문 ID, 고객사 ID, 주문 날짜, 품목, 수량, 상태, 특이사항 메모, 긴급 플래그, 수정 일시, 취소 일시
- **FR-01-02: 주문 상태 추적** — 주문은 접수, 확인, 출고, 배송, 완료의 상태를 가진다.
  - 화면 흐름: Step 1: 주문 접수 → Step 2: 주문 상태 변경 (확인, 출고, 배송, 완료)
  - 비즈니스 로직: 주문 상태는 순차적으로 변경되며, 각 상태 변경 시 로그가 기록된다.
  - 데이터 항목: 주문 ID, 현재 상태, 상태 변경 일시
- **FR-01-03: 대량 주문 엑셀 업로드** — 고객사는 엑셀 파일을 업로드하여 대량 주문을 처리할 수 있다.
  - 화면 흐름: Step 1: 엑셀 파일 다운로드 (템플릿) → Step 2: 엑셀 파일 작성 → Step 3: 엑셀 파일 업로드 → Step 4: 업로드 결과 확인
  - 비즈니스 로직: 엑셀 파일의 각 행은 하나의 주문으로 처리되며, 파일 내 오류가 있을 경우 해당 행은 처리되지 않고 오류 메시지가 반환된다.
  - 데이터 항목: 엑셀 파일 (주문 ID, 고객사 ID, 품목, 수량 등)

## 1. DB 스키마

### 주문 테이블 (Orders)
```sql
CREATE TABLE Orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    order_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    item VARCHAR(255) NOT NULL,
    quantity INT NOT NULL CHECK (quantity > 0),
    status VARCHAR(50) NOT NULL CHECK (status IN ('접수', '확인', '출고', '배송', '완료', '취소')),
    notes TEXT,
    urgent_flag BOOLEAN NOT NULL DEFAULT FALSE,
    modified_at TIMESTAMP,
    canceled_at TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES Customers(customer_id)
);

CREATE INDEX idx_orders_customer_id ON Orders(customer_id);
CREATE INDEX idx_orders_status ON Orders(status);
```

### 주문 상태 변경 로그 테이블 (OrderStatusLogs)
```sql
CREATE TABLE OrderStatusLogs (
    log_id SERIAL PRIMARY KEY,
    order_id INT NOT NULL,
    previous_status VARCHAR(50) NOT NULL,
    new_status VARCHAR(50) NOT NULL,
    changed_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES Orders(order_id)
);

CREATE INDEX idx_order_status_logs_order_id ON OrderStatusLogs(order_id);
```

## 6. 수용 기준 (테스트 케이스 수준)

- **Given**: 고객사가 주문을 접수하려고 할 때
  - **When**: 모든 필수 필드를 올바르게 입력하면
  - **Then**: 주문이 성공적으로 접수되고 상태는 '접수'로 설정된다.

- **Given**: 고객사가 주문을 수정하려고 할 때
  - **When**: 주문 상태가 '확인'인 경우
  - **Then**: 주문 정보가 성공적으로 수정된다.

- **Given**: 고객사가 주문을 취소하려고 할 때
  - **When**: 주문 상태가 '접수'인 경우
  - **Then**: 주문이 성공적으로 취소되고 상태는 '취소'로 변경된다.

- **Given**: 고객사가 대량 주문 엑셀 파일을 업로드하려고 할 때
  - **When**: 파일 형식이 올바르고 모든 행의 데이터가 유효하면
  - **Then**: 모든 주문이 성공적으로 처리된다.

## 1. DB 스키마

### 테이블: Inventory
- `item_id` (VARCHAR, NOT NULL, PRIMARY KEY)
- `current_quantity` (INT, NOT NULL)
- `warehouse_location` (VARCHAR, NOT NULL)

```sql
CREATE TABLE Inventory (
    item_id VARCHAR PRIMARY KEY,
    current_quantity INT NOT NULL,
    warehouse_location VARCHAR NOT NULL
);
CREATE INDEX idx_inventory_item_id ON Inventory(item_id);
```

### 테이블: StockHistory
- `history_id` (SERIAL, PRIMARY KEY)
- `item_id` (VARCHAR, NOT NULL, FOREIGN KEY REFERENCES Inventory(item_id))
- `quantity` (INT, NOT NULL)
- `timestamp` (TIMESTAMP, NOT NULL DEFAULT CURRENT_TIMESTAMP)
- `transaction_type` (VARCHAR(10), NOT NULL CHECK (transaction_type IN ('IN', 'OUT')))
- `responsible_person_id` (VARCHAR, NOT NULL)

```sql
CREATE TABLE StockHistory (
    history_id SERIAL PRIMARY KEY,
    item_id VARCHAR NOT NULL REFERENCES Inventory(item_id),
    quantity INT NOT NULL,
    timestamp TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    transaction_type VARCHAR(10) NOT NULL CHECK (transaction_type IN ('IN', 'OUT')),
    responsible_person_id VARCHAR NOT NULL
);
CREATE INDEX idx_stockhistory_item_id ON StockHistory(item_id);
```
